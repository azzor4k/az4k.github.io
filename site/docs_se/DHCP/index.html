    TỔNG QUAN VỀ DHCP

    Khái niệm

    • DHCP viết tắt của Dynamic Host Configuration Protocol
    • Mục đích là cung cấp các thông tin như IP addresss, subnet mask, gateway, dns server,...cho tất cả các máy tính thiết bị đang kết nối qua thiết bị mạng.
    • Máy chủ đảm nhận hoạt động kiểm tra thông tin IP,.. và thực hiện trao đổi cấp phát IP động thông qua giao thức DHCP.

    DHCP server bao gồm

    • Options: Bao gồm thông tin như IP address. subnetmask, Gateway, DNS server. Toàn bộ thông tin sẽ được DHCP server cung cấp đầy đủ.
    • Scope: Một dãy hay một đoạn địa chỉ IP đã được quy hoạch và chỉ định rỏ ràng cho DHCP server được cấp phép IP động thông qua DHCP cho máy client.
    • Reservation: Là những thông tin IP address backup. Cấu hình sẵn IP address đó cho một số lại máy tính client.
    • Lease: Time for rent hoặc tồn tại của địa chỉ IP động đối với mỗi client.

    Cách thức hoạt động

    Quá trình của DHCP bao gồm 4 bước:

    B1: Client gửi gói tin DHCPDISCOVER (broadcast) trên lớp mạng thiết bị -> server. Mục đích của DHCPDISCOVER là tìm được DHCP Server cấp IP.

    B2: DCHP server nhận gói tin DHCPDISCOVER của client. DHCP server tiến hành query Database IP đã cấp cho client khác còn trống IP nào không. Nếu có sẽ tiến hành gửi thông tin IP kèm các thông tin Options trong gói tin DHCPOFFER (unicast) -> client

    B3: Client nhận được gói tin DHCPOFEER phản hổi từ DHCP Server. Sẽ tiến hành trả lời DHCP Server bằng gói tin DHCPREQUEST (broadcast) với nội dung là thông tin địa chỉ IP và Options trong gói tin DHCPOFFER trước đó.

    B4: DHCP server đồng ý cấp IP cho client thông qua gói DHCPPACK (unicast)

    Tổng kết

    • DHCP SERVER (options, scope, reservation, lease)
    • Client [DHCPDISCOVER][broadcast] -> server [DHCPOFFER][unicast] -> client [DHCPREQUEST][broadcast] -> [DHCPPACK][unicast] -> client.
