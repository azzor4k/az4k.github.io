    JVM - Java Virtual Machine

    Tổng Quát

    • Tất cả các chương trình khi muốn thực thi được thì phải bắt buộc phải được biên dịch sang code máy.
    • Code máy từng loại kiến trúc CPU của mỗi máy tính là hoàn toàn khác nhau.
    • Ví dụ trước đây một chương trình chạy được trên Windows muốn chạy được trên hệ điều hành khác chẳng hạn như linux thì phải chỉnh sửa và biên dịch lại đó là một bất tiện khó khăn đối với các lập trình viên.
    • Từ đó ngôn ngữ lập trình Java ra đời, nhờ vào máy ảo Java mà khó khăn trên đã được giải quyết một cách dễ dàng.
    • Với một chương trình được viết bằng ngôn ngữ Java sẽ được biên dịch ra code của máy ảo Java (code java bytecode).
    • Sau đó máy ảo Java sẽ chịu trách nhiệm chuyển tiếp code java bytecode thành code máy tương ứng.

    Định Nghĩa

    JVM viết tắt của Java Virtual Machine 

    Là một máy ảo Java- trình thông dịch của Java.
Là môi trường chạy các ứng dụng được viết bằng ngôn ngữ JAVA.
Nó cung cấp môi trường runtime mà trong đó Java Bytecode có thể thực thi được.
Nhờ có JVM mà Java có thể chạy trên nhiều Platform khác nhau. 
Cứ mỗi Platform sẽ có một JVM tương ứng. VD: JVM cho Ubuntu. JVM cho Windows.
    Bytecode là 
    cách thức lưu trữ dạng mã các chỉ thị trong lập trình máy tính, 
được thiết kế để phần mềm thông dịch thực hiện trên nền tảng máy ảo.
    Các nhiệm vụ chính của JVM - Tải code - Kiểm tra code - Thực thi code - Cung cấp môi trường runtime

    Cấu Trúc

    Các thành phần chính của JVM

    • Classloader: là một hệ thống con của JVM, được sử dụng để tải class file hay lớp được định nghĩa.
    • Class Area: lưu trữ cấu trúc của các lớp, thuộc tính, phương thức của lớp, code của cúa phương thức.
    • Heap: là vùng lưu trữ dữ liệu runtime mà các đối tượng được khởi tạo
    • Stack: lưu trữ các fame. Mỗi fame chứa các biến cục bộ và thực thi một hàm gọi và trả về kết quả. Mỗi thread có một stack riêng được tạo tại cùng thời điểm với thread. Mỗi frame sẽ được tạo khi một hàm được gọi và hủy khi việc gọi hàm kết thúc.
    • Program counter Reister: nó chứa địa chỉ của máy chủ ảo chỉ lệnh JVM hiện tại đang thực thi.
    • Native method stack: bao gồm các phương thức được thực thi sử dụng trong chương trình
    • Execution engine:
      • Bao gồm bộ xử lý ảo
      • Trình thông dịch: đọc byte code và thực hiện các chỉ thị.
      • JIT Compiler: biên dịch mã byte code sang mã máy.
    • Native method interface: là khung lập trình cho phép các phương thức chạy trên JVM
    • Java Native Libraries: là thư viện cho phép được viết bằng các ngôn ngủ khác như C, C++, Assembly trên JVM.

    Cơ chế làm việc

    JVM được chia thành 3 phần chính: 

    1. Class-Loader Subsystem: sẽ tìm kiếm và tải các file .class vào vùng nhớ của Java.
2. Runtime Data Area: là vùng nhớ cấp phát cho JVM
3. Execution Engine: chuyển các lệnh của JVM trong file .class thành các lệnh của máy.

    Hoạt động

    • Cung cấp môi trường để code java có thể thực thi.
    • Chương trình Java khi biên dịch sẽ tạo ra các file *.class chứ bytecode.
    • Đọc bytecode stream.
    • Kiểm tra bytecode
    • Các file *.class này sẽ được JVM thực hiện chuyển bytecode thành mã máy tương ứng với từng hệ điều hành và phần cứng khác nhau để thực thi.
