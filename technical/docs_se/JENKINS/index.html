    Jenkins

    Tổng quan về Jenkins

    Jenkins là phần mềm tự động hóa, mã nguồn mở và viết bằng JAVA.

    Là một open source continuous integration server. Build và test phần mềm một cách liên tục và theo dõi trạng thái của các remote jobs. Giúp team member và user thường xuyên có được code chạy ổn định.

    Jenkins thường sử dụng cho: - Building một project - Chạy thử nghiệm để phát hiện lỗi và các vấn đề khác ngay khi chúng được thông báo. - Phân tích static code - Và deloy triển khai

    Thực hiện công việc lặp đi lặp lại, tiết kiệm thời gian và tối ưu hóa quá trình phát triển với Jenkins

    Jenkins giúp tự động hóa các quy trình phát triển phần mềm, hiện nay được gọi theo thuật ngử là tích hợp liên tục và còn dùng đến trong việc phân phối liên tục.

    Jenkins là phần mềm dạng server, chạy trên nền servlet với sự hổ trợ của Apache tomcat.

    Servlet là gì ? 

    Servlet có thể được mô tả bằng nhiều cách, tùy vào nhiểu ngữ cảnh:
- Là một công nghệ được sử dụng tạo ra ứng dụng Web.
- Là một API cung cấp các interface và các lớp bao gồm tài liệu.
- Là một thành phần web được triển khai trên máy chủ để tạo ra trang web động.
    Kiến trúc servlet?

    Apache tomcat là gì? 

    Là một Java servlet được phát triển bởi Apache Software Foundation.
Java servlet API cho phép nhà ứng dụng thêm những nội dung động vào Webserver dưới nền tản Java.

    Jenkins hỗ trợ hầu hết các phần mềm mã nguồn phổ biến hiện nay như Git,Subversion,...

    Jenkins việc kích hoạt build dự án phần mềm có thể được thực hiện bằng nhiều cách:

        + Dưạ theo các lần commit trên mã nguồn.

    + Theo các khoảng thời gian.

    + Kích hoạt qua các URL.

    + Kích hoạt qua các job khác được build.

    Jenkins chức năng của nó có thể mở rộng thông qua các plugins.

    Plugins ? - Plugin là phần mở rộng trong Jenkins, hỗ trợ các dự án không dùng ngôn ngữ Java - Plugin có thể tích hợp hầu hết các hệ thống quản lý mã nguồn và cơ sở dữ liệu vào Jenkins. - Plugin có thể thay đổi giao diện và thêm chức năng mới cho Jenkins.

    Tích hợp liên tục - Continous Integration

    • Các developer thực hiện các ticket và đẩy code lên thư mục nơi sẽ lưu trữ code của project.
    • Hệ thống CI sẽ lắng nghe và thực hiện pull code về, build rùi test code.
    • Trong trường hợp build, test thất bại, các thành viên trong team sẽ nhận ra các lỗi phát sinh và các developer sẽ chịu trách nhiệm sữa lỗi và đẩy code sữa lổi đó lên repo.

    Vai trò của Jenkins trong hệ thống tích hợp liên tục

    Là một ứng dụng web application mã nguồn mở đóng vai trò là máy chủ build & test của hệ thống tích hợp liên tục.

    Jenkins giữ vai trò quan trọng trung tâm là CI server (build & test server) cho mọi hoạt động của hệ thống.

    • Nhờ vào việc hỗ trợ đa nền tảng, công nghệ Jenkins giúp khép kín quá trình phát triển phần mềm một cách tự động theo mô hình Agile nói chung và hệ thống tích hợp liên tục nói riêng.
    • Với hơn 400 plugin và được phát triển thêm không ngừng Jetkins đã trở thành trợ thủ đắc lực cho hệ thống CI.

    Nói chung jetkins tất cả với nó hầu như là tự động. => Giảm thời gian và chi phí.

    Tiến hành tạo log file cho Jenkins khi build như sau:

    Chọn Configue, vào phần Build Chọn Execute shell nhập vào Command như trong hình sau:

    Trong máy cài đặt Jenkins tiến hành tạo file log và nơi lưu trữ như sau: 

    $ sudo mkdir /etc/Jenkins_Test
$ sudo mkdir /etc/Jenkins_Test/Log
$ sudo vi /etc/Jenkins_Test/create_log.sh
    Nội dung file create_log.sh như sau: 
    #!/bin/sh
day=$(date +%d-%m-%Y)
FILE_NAME=/etc/Jenkins_Test/Log/log_$day.log
DATE_NOW=$(date +%Y/%m/%d-%H:%M:%S)
touch $FILE_NAME
echo "Creating txt file"
echo "Created by LNT. Created time is $DATE_NOW" >>  $FILE_NAME
echo "Completed"
    Cấp quyền thực thi cho file log cũng như thư mục lưu trữ: 
    $ sudo chmod +x /etc/Jenkins_Test/create_log.sh
$ sudo chown jenkins.jenkins /etc/Jenkins_Test
$ sudo chown jenkins.jenkins /etc/Jenkins_Test/Log
